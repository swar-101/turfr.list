## Contributing to turfr™

Thanks for contributing.


## Workflow

1. Create a branch from `dev`
2. Make your changes
3. Open a Pull Request to `dev`
4. Follow the PR template
5. Get approval
6. Merge using **Squash & Merge**


## Branch Naming

Use clear, descriptive names:

* `feat/payment-upi-flow`
* `fix/waitlist-scroll`
* `refactor/match-layout`


## Pull Request Guidelines

All PRs should follow this structure:

### Summary

Explain what this PR does.

### Problem

What issue or limitation existed?

### Solution

How did you solve it?

### Result

What improved or changed?

### Notes (optional)

Any additional context.

### Screenshots (optional)

Include screenshots for UI changes to demonstrate what changed.

Keep PRs focused and easy to review.

## Rules

* No direct commits to `main` or `dev`
* All changes must go through PR
* Use meaningful branch names and PR titles
* Keep changes small and intentional


## Code Quality

* Write clear, readable code
* Prefer simplicity over cleverness
* Avoid unnecessary dependencies
